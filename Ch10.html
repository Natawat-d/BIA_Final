New AI-Based Trends in Analytics and Data Science

Key idea

Core message: Analytics is being reshaped by where computing happens (the cloud), what counts as data (location, images, sensors — "alternative data"), and what is connected (the IoT). The unifying chain: things → sensors → data → analytics → knowledge → decisions/actions. IoT is not an application — it is the infrastructure/platform that feeds analytics.

Opening vignette — Siemens avoids train failures

Analysis of sensor data helps Siemens avoid train failures. Sensors on trains stream condition data continuously; analytics predicts failures before they happen (predictive maintenance) and optimises asset utilisation. The discussion questions push you to think about which parameters to measure, how weather data helps, and — the memorable arithmetic — how much data 1,000 sensors at 1 KB/second generate in a month (≈ 1,000 × 1 KB × 2.6 M seconds ≈ 2.6 TB/month per train) and how you would store it.

Second case — "Discover Foods": IoT + machine learning maintain food quality. Temperature is critical, and quality degrades when temperature fluctuates (repeated rises/falls in the time-series). Remote sensing + cloud + ML automate the real-time upkeep of temperature, microorganism growth and humidity → better quality, lower operating cost, less spoilage.

Cloud computing

Definition: "a model for enabling convenient, on-demand network access to a shared pool of configurable computing resources (networks, servers, storage, services) that can be rapidly provisioned and released with minimal management effort or service-provider interaction."

  • A new name for older trends: utility computing, application service provider, grid computing, on-demand computing, SaaS — and, going further back, centralised computing with dumb terminals. The name comes from drawing the Internet as a "cloud."
  • Examples: web-based e-mail (storage + processing on the provider's hardware, centralised control/upgrades); AWS (Amazon's infrastructure for e-commerce, BI, CRM, SCM); Salesforce, IBM Cloud, Microsoft Azure, Google, Adobe.
  • Why it matters for analytics: cost-effective, scalable data warehousing; rapid diffusion of advanced tools without big up-front technology investment.
  • Service-oriented DSS = a cloud-based analytics system. Four components can be bought individually or bundled as a service: operational systems · data warehouses · OLAP · end-user components.

Virtualization — the essential enabling technology

Creation of a virtual version of an operating system or server, in three areas:

  • Network virtualization — split available bandwidth into channels, allocate to a server/device.
  • Storage virtualization — pool physical storage from many devices into a single logical storage device.
  • Server virtualization — mask physical servers from users → centralise & simplify management.

Deployment models, platforms & security concerns

  • Private cloud — operated solely for/by a single organisation. Public cloud — shared, offered by providers over the Internet. Hybrid cloud — a mix of both.
  • Platform providers: Amazon Elastic Beanstalk · IBM Cloud · Microsoft Azure · Google App Engine · OpenShift (Red Hat's PaaS).

Challenges & the top 3 security threats

Major challenges: security, service level, data governance, loss of control and privacy, legal liabilities, cross-border political issues.

Top three security threats (Cloud Security Alliance): ① data loss and leakage, ② hardware failure of equipment, ③ insecure interface.

Cloud analytics in action: Zion China uses Azure IoT + Stream Analytics + Machine Learning + Power BI for a mobile health-monitoring tool (glucose, blood pressure, diet, medication, exercise). Chime (digital bank) uses the Snowflake elastic data warehouse to pull from 14 data sources for personalised banking. A West-Coast utility uses cloud-mobile tech for real-time incident reporting.

Location-based analytics

Incorporating location into analytics — static (geospatial) vs dynamic (real-time) location data; for organisations vs for individuals/consumers.

Geospatial analytics

Adds the dimension of "where" to analyses that already answer who, what, when, how much. Data comes from GIS (Geographic Information Systems); market leader = ESRI with ArcGIS. Applications: agriculture, crime analysis, disease-spread prediction; overlay weather/environmental effects for richer forecasting and planning.

Real-time location intelligence

Cars, buses, taxis, phones, cameras and navigation devices constantly broadcast location via GPS, Wi-Fi and cell-tower triangulation. Analysing large-scale movement patterns identifies distinct classes of behaviour → better promotions and pricing. Augmented reality is an extension of location-based analytics (location + camera images + sensors → immersive experience; a big push at Microsoft and Apple).

Consumer applications

Waze — social navigation app: alerts on accidents, police checkpoints, speed traps, construction from other users' inputs. ParkPGH (Pittsburgh, with Carnegie Mellon) — predicts parking availability. Consumers use analytics "without ever realizing it." Key concern: loss of privacy (→ Ch.11).

Cases: home-loan appraisal improved with BI + GIS; Starbucks exploits GIS + analytics (demographics, store-location siting, the mobile app, in-store Wi-Fi) to grow worldwide.

Image analytics & alternative data

High-resolution cameras + cheap storage + deep learning (Ch.7's CNNs) make images a serious data source. Satellite data = "alternative data" — a form of Big Data:

  • World Bank — satellite data → strategic recommendations for urban planners in developing nations.
  • EarthCast — weather updates for a large US airline from 60 government-operated satellites.
  • Imazon — detects Amazon deforestation. Global Fishing Watch — spots where and when illegal fishing happens.
  • Case 10.5: image analysis estimates plant/foliage cover on farms (an app beats a visual check).

Internet of Things — essentials

IoT = a computerised network connecting many objects (people, animals, devices, sensors, buildings, items), each with an embedded microprocessor, connected wirelessly via the Internet, exchanging data with each other and their environment anytime, anyplace (ubiquitous computing).

Kevin Ashton (who coined the term): "sensors connected to the Internet and behaving in an Internet-like way by making open, ad hoc connections, sharing data freely, and allowing unexpected applications, so computers can understand the world around them and become humanity's nervous system."
Textbook working definition: a network of connected computing devices; each object has a unique identifier (UID) and can collect and transfer data automatically across the network.

Characteristics (Miller, 2015) — 5 to memorise

  1. Large numbers of objects ("things") can be connected.
  2. Each thing has a unique ID (IP address).
  3. Each thing can receive, send and store data automatically.
  4. Each thing is delivered mostly over the wireless Internet.
  5. Each thing is built on machine-to-machine (M2M) communication.

The Internet connects people to each other; IoT connects things to each other and to the sensors that collect data.

IoT platforms: Amazon AWS IoT · Microsoft Azure IoT Suite · Predix (GE) · IBM Watson IoT · Teradata Unified Data Architecture.

How IoT works — the exam-critical chain

IoT is NOT an application — it is an infrastructure/platform/framework that supports applications. The process: the ecosystem contains many things → sensors collect information → the information is displayed, stored and processed analytically (e.g. data mining) → analysis converts information into knowledge/intelligence → expert systems or machine learning turn knowledge into decision support (by people and/or machines) → improved actions and results → new applications and use cases.

Sensors and RFID

A sensor = an electronic device that automatically collects data about events in its environment. Sensors are a critical part of IoT (and of robotics and autonomous vehicles). They have distance and precision limits; very-short-range sensors are called proximity sensors and are more reliable than longer-range ones. In large-scale applications, sensors send data to the cloud for processing.

RFID (Technology Insight 10.1)

  • RFID = uses radio-frequency waves to identify objects; part of the automatic-identification family alongside barcodes and magnetic strips, but stores richer identification data.
  • Works with tags and readers; active tags (long range, powered) vs passive tags (short range).
  • Spread through retailers' supply chains; combined with other sensors it is central to IoT.
Case 10.7 — Rockwell Automation: monitors expensive oil & gas exploration assets to predict failures. Exam-style question from the slides: does this fit the three Vs (volume, variety, velocity) of Big Data? (Yes — high-frequency multi-sensor streams.)

IoT applications — smart homes, factories, cities

  • French National Railway (SNCF) — manages 15,000 trains over 30,000 km for ~14 million passengers, using IBM Watson in the background for quality, availability and safety.
  • Smart home — automated, interconnected lights, appliances, security and entertainment → comfort, security, low energy cost, convenience. Most existing homes aren't smart but can be cheaply given partial smartness. Smart appliances can be remotely controlled per user preference: learning thermostat, smoke detector/alarm, video surveillance, Samsung smart fridge, LG InstaView, Whirlpool smart washer.
  • Barriers to smart-home adoption (the 3 C's): Compatibility (too many vendors/products to connect), Communication (consumers disagree on what a smart home even is), Concentration (brands must target the most interested segments, e.g. Gen Y) — plus cost justification, invasion of privacy, security, ease of use.
  • Smart digital factory — a flexible system that can self-optimise performance across a network, self-adapt to and learn from new conditions in (near) real time, and autonomously run entire production processes.
  • Smart cities — transportation is the major improvement area (too many vehicles). Examples: Valerann (Israeli start-up) smart road studs; Hong Kong's Smart Mobility Consortium.
  • Simple IoT examples: autonomous vehicles, smartbins, a fridge that reorders food, Clorox's Brita filter that auto-orders its replacement. In production/logistics IoT enables real-time tracking of everything, route/logistics optimization, condition assessment of critical objects, and monitoring of perishables.

Autonomous vehicles & 5G

Autonomous (self-driving) vehicles — driverless/robot-driven cars. Google started it, Tesla made it famous; most manufacturers are investing; France, Singapore, China and others already run them. Implementation issues: cost of real-time 3D map technologies · AI software must be nimbler · readying automakers and insurers · sensor technology needs better cost-efficiency · IoT systems and connectivity must improve.

5G — "catalyst for the Fourth Industrial Revolution"

  1. Very high-speed access, even in densely populated areas.
  2. Connects everything — supports massive IoT and M2M communication.
  3. Real-time latency — minimal network delay, enabling new vertical-industry services.
  4. Significant improvements to security and privacy.
  5. A service-based architecture (SBA) → more flexibility and service diversity.

Other emerging AI topics

Robotic Process Automation (RPA)

Technology that reduces employee workload by automating repetitive, high-volume process steps. Software robots ("bots") replicate the actions of human workers, freeing humans for more important activities. Gains: productivity ↑, human errors eliminated, operational cost ↓, labour-intensive tasks automated, work completed faster.

Bioinformatics & network science

Bioinformatics and health analytics are major AI growth areas. Network Analysis (NA) analyses complex problems involving interactions among features or observations; used with deep learning and NLP. Example: comorbidity networks for health-disparity analysis, and (Case 10.11) predicting hospital length of stay using comorbidity networks. Network analysis spans all three analytics echelons — descriptive, predictive and prescriptive; as everything becomes connected, network science keeps gaining ground.

Other recent developments

  • Web 3.0 — a decentralized structure: blockchain, cryptocurrency, NFTs.
  • Metaverse (a virtual world where objects interact) and digital twins — a digital representation of a real-world product/system/process, an effectively indistinguishable digital counterpart.
  • GPT-3/ChatGPT (OpenAI) · LaMDA (Google's dialogue model) · BlenderBot (Meta's chatbot).
