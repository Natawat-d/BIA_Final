Key idea

Core message: Deep learning is the newest member of the AI/ML family. Its superpower — and the reason it beats classic ML on images, text, and speech — is automatic feature extraction and representation learning: the network learns its own features from raw data instead of relying on hand-engineered ones. "Deep" simply means many hidden layers, trained on large data with GPUs.

Chapter arc: shallow ANN mechanics → training (backpropagation) → deep architectures for different data types (MLP for tabular, CNN for images/text, RNN/LSTM for sequences) → implementation frameworks → cognitive computing (IBM Watson) as the applied, human-augmenting face of these technologies.

Opening vignette — fighting fraud with deep learning (Danske Bank)

Business problem: bank fraud — using deceptive, illegal methods to steal assets from financial institutions or their customers, increasingly through digital channels. Typical fraud types:

  • Phishing — tricking individuals into revealing credentials.
  • Identity theft — using stolen personal information to open accounts or take over existing ones.
  • ATM skimming — stealing card data via compromised ATMs.
  • Loan fraud — intentionally providing false information to obtain loans.

Solution & result: Danske Bank replaced rule-based/traditional detection with deep learning. Measured on accuracy and the ROC curve, deep learning outperformed traditional ML techniques — fewer missed frauds and fewer false alarms.

Discussion point the slides flag (exam-friendly): the false-positive vs false-negative trade-off (Type I vs Type II error). A false positive blocks a legitimate customer transaction (friction, complaints); a false negative lets real fraud through (direct loss). Where you set the alert threshold is a business decision, not just a statistical one.

Basics of "shallow" neural networks

Artificial Neural Networks (ANN) are abstractions of the human brain's biological network of neurons. An artificial neuron is called a processing element (PE).

The neuron's arithmetic

A neuron with R inputs computes a weighted sum plus a bias, then passes it through a transfer (activation) function:

n = w₁,₁p₁ + w₁,₂p₂ + … + w₁,RpR + b  → in vector form  n = wp + b, output a = f(n)

Elements of an ANN (Technology Insight 7.1)

  • Processing element (PE) — the neuron itself.
  • Network structure — how neurons are organised: input layer → hidden layer(s) → output layer.
  • Connection weights — the adjustable parameters; they store what the network has learned.
  • Summation function — combines multiple incoming signals into a single value.
  • Transfer (activation) function — translates the summed input into the neuron's output; common choices: sigmoid (0–1), tanh, ReLU (rectified linear — the deep-learning default), linear, hard limit.
Vocabulary check: a "typical" small network in the slides has three layers and eight neurons — the point is that a shallow network has one (maybe two) hidden layers; a deep network has many.

How an ANN learns

Development process (iterative, with constant feedback): collect data → separate into training/test sets → define network structure → select a learning algorithm → set parameters & initialise weights → transform data → train → test → implement — looping back whenever results disappoint.

The learning loop (3 steps)

1 Compute temporary outputs→2 Compare with desired targets→3 Adjust weights & repeat

Backpropagation (the standard training algorithm)

  1. Initialise weights with random values.
  2. Read in the input vector and the desired output.
  3. Compute the actual output via forward calculation.
  4. Compute the error (desired − actual).
  5. Change the weights by working backward from the output layer through the hidden layers.

Repeat over many examples/epochs until error is acceptably low.

Illuminating the ANN black box

ANN are typically criticised as black boxes — accurate but opaque. The chapter's remedy: sensitivity analysis based on leave-one-out — perturb or omit each input variable in turn and measure how much the trained network's output changes; the change ranks each variable's importance.

Analytics in Action 7.2 — injury severity in traffic accidents: eight binary ANN models were built and sensitivity analysis ranked the factors driving injury severity. Why use a black-box ANN over transparent logistic regression? Because the ANN captures non-linear relationships and predicts better — and sensitivity analysis restores (some) interpretability afterwards.

Deep neural networks — what makes them "deep"

  • Deep = more hidden layers (not just more neurons in one layer). Empirically, adding layers tends to help more than merely widening a layer.
  • Hardware: beyond CPUs, deep learning uses GPUs programmed with CUDA (NVIDIA's Compute Unified Device Architecture); Google built TPUs (Tensor Processing Units).
  • Data: needs large datasets; inputs are tensors = N-dimensional arrays (e.g. a colour image is a 3-D tensor: height × width × RGB channels).
  • Different deep-network types suit different tasks — MLP (general), CNN (images/text), RNN/LSTM (sequences).

Feedforward MLP-type deep networks

The multilayer perceptron (MLP) is the most common deep network: fully connected layers passing vectors forward. Two training subtleties from the slides: random initial weights can strongly affect where training ends up, and pretraining network parameters (initialising from an unsupervised pass) measurably improves classification results.

Analytics in Action 7.3 — Georgia DOT variable speed limits: data science on traffic-flow data to set variable speed limits and relieve congestion — an example of deep/predictive analytics embedded in public-sector operations.

Convolutional neural networks (CNN)

The most popular deep-learning method, used for image/video processing and text recognition. Defining property: at least one convolution layer.

  • Convolution layer — a small kernel (filter), e.g. 2×2, slides across the input matrix computing local weighted sums; this detects local features (edges, corners, textures) and shares weights, drastically reducing parameters vs a fully connected layer.
  • Pooling (sub-sampling) layer — consolidates large tensors into smaller ones, reducing the number of model parameters while keeping only the important features (e.g. max pooling). Several pooling types exist.
  • Typical unit: convolution → pooling → convolution → pooling → … → fully connected → output.

Image processing milestones

  • ImageNet (image-net.org) — the benchmark dataset that powered the CNN revolution.
  • AlexNet — the landmark CNN architecture for image classification.
  • GoogLeNet — introduced the Inception feature (parallel multi-size convolutions).
  • Google Lens — a consumer service built on convolutional deep networks for image recognition.
  • Face recognition (Analytics in Action 7.4) — technically challenging (pose, lighting, ageing) and socially contentious (privacy, surveillance, bias).

Text processing with CNN

Word embeddings (Google's word2vec) map words into a vector space where semantically similar words sit near each other; CNNs over embedded text handle tasks such as relation extraction in text mining.

Recurrent neural networks (RNN) & LSTM

RNN are designed to process sequential inputs (language, time series, speech): the recurrent unit feeds its output back into itself, so the network has memory of what came before.

LSTM (long short-term memory) is the practical RNN variant. In a dynamic network the weights are the long-term memory while the feedback loop plays the short-term memory role; LSTM adds gates that decide what to keep, forget, and output — solving the vanishing-gradient problem on long sequences.

Flagship application: the Google Neural Machine Translator (GNMT) — LSTM-based machine translation with close-to-human performance. Other LSTM uses: speech recognition, handwriting, time-series forecasting.

ChatGPT & large language models

  • ChatGPT (OpenAI) is the chapter's example of a large language model (LLM) — and the fastest AI technology adoption to date.
  • Applications: customer service, healthcare, marketing, computer programming, and more.
  • Limitations (know these): biases, false answers (hallucination), obsolete training data.
  • How it works (high level): a transformer-based network pre-trained on massive text to predict the next token, then fine-tuned with human feedback.

Cognitive computing & IBM Watson

Definition: systems that use mathematical models to emulate (or partially simulate) the human cognition process to find solutions to complex problems and situations where the potential answers can be imprecise. The icon: IBM Watson winning Jeopardy!

How cognitive computing works — four attributes

  • Adaptive — learns as information and goals change.
  • Interactive — converses naturally with users (and other systems).
  • Iterative and stateful — remembers earlier interactions, asks clarifying questions when a problem is ambiguous.
  • Contextual — draws meaning from context (domain, time, location, user).

Underlying technologies: data mining, pattern recognition, deep learning, and NLP — mimicking the way the human brain works.

Use cases & cognitive search

Use cases: smart adaptive search engines, NLP, speech recognition, language translation, context-based sentiment analysis, face/emotion recognition, risk assessment, fraud detection, behavioural recommendations. Cognitive search goes beyond keyword search: handles a variety of data types, contextualises the search space, employs advanced AI, and lets developers build enterprise-specific applications.

Applications named in the slides: Watson predicting the Eurovision Song Contest winner; MYCIN — the 1980s rule-based expert system (bridge to the Expert Systems lecture); plus healthcare/medicine, security, finance, retail, education, government, research.
